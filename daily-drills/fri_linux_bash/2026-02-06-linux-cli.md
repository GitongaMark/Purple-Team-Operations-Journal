## 3. Advanced File Hunting (The `find` Command)
The most powerful tool mastered today was `find`.

### Scenario A: Filtering by Content Type
```bash
find . -type f -exec file {} + | grep "text"
```
* **Logic:** Finds all files (`-type f`), runs the `file` command on them (`-exec`), and filters for "text".

* **Use Case:** Locating readable config files amidst a pile of binaries.

### Scenario B: The "Needle in a Haystack"

```bash
find . -type f -size 1033c ! -executable
```
* **Logic:** Finds a file that is exactly 1033 bytes (`c`) AND is NOT executable (`!`).

* **Use Case:** Identifying a specific payload or log file based on known attributes.

### Scenario C: Searching the Whole System (Permission Handling)
```bash
find / -user bandit7 -group bandit6 -size 33c 2>/dev/null
```
* **Key Concept:** `2>/dev/null`

* **Why:** When searching root directories (`/`), you get hundreds of "Permission Denied" errors. Redirecting Stderr (`2>`) to the null device cleans the output so you only see the success hits.

## 4. Handling "Edge Case" Filenames
Linux allows filenames that are difficult to reference. I learned how to handle them:

*   **Files named - (Dash):** The system interprets `-` as "Standard Input." Fix: Use the relative path `./-` or redirect input `cat < -`.
*   **Files with Spaces:** Fix: Use quotes `cat "file name"` or escape characters `cat file\ name`.

## 5. Key Takeaway
The Linux shell treats everything as a file or a stream. Mastering `find` allows me to locate threats based on metadata (who owns it, how big is it) rather than just filename, which is critical for finding malware that disguises itself.
